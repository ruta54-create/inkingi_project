# SokoHub

Minimal Django marketplace (SokoHub) — vendors add products, customers buy.

## Quick start (development)

1. Create a virtual environment and activate it (Windows PowerShell):

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

2. Install dependencies:

```powershell
pip install -r requirements.txt
```

3. Copy environment example and update secrets:

```powershell
copy .env.example .env
# Edit .env and set SECRET_KEY and DEBUG as needed
```

4. Run migrations and create a superuser:

```powershell
python manage.py migrate
python manage.py createsuperuser
```

5. (Optional) Load sample data:

```powershell
python scripts\seed_sample_data.py
```

6. Run the development server:

```powershell
python manage.py runserver
```

7. Open http://127.0.0.1:8000/

## Environment variables

- `SECRET_KEY` — Django secret key (required in production)
- `DEBUG` — `True` or `False`
- `ALLOWED_HOSTS` — comma-separated allowed hosts
- `DATABASE_URL` — optional, for using Postgres or another DB
- Email configuration variables (optional): `EMAIL_HOST`, `EMAIL_PORT`, `EMAIL_HOST_USER`, `EMAIL_HOST_PASSWORD`, `EMAIL_USE_TLS`

## Development notes

- Django app uses `AUTH_USER_MODEL = 'accounts.CustomUser'`.
- Migrations are tracked under each app's `migrations/` directory — ensure these are committed to version control.

## Committing migrations

Migrations generated by `makemigrations` should be added and committed. Example:

```powershell
git add .
git commit -m "Add migrations and docs"
```

## Troubleshooting

- If you see `DisallowedHost`, add the host to `ALLOWED_HOSTS` in your `.env` / settings.
- If product images don't appear, ensure `MEDIA_URL`/`MEDIA_ROOT` are configured and the dev server is serving media in `DEBUG` mode.
